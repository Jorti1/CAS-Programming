Let's begin our first project.

In this project, I will create a step by step guide for the pythagorean theorem.

1) From the calculator menu, we will navigate to the "New option"





2) Click and navigate to "Add program editor" and create new


3) Name your program, select the type of program, and the library access. We will name our program "abc" since that is the same variables as the pythagoream theorem. In this instance, our type will be program since using the other option, function, will be limative by not allowing us to define global variables. Library access simply means if you want the program to show in your catalog. For testing purposes, it should be set to private or none to avoid using a broken program. Fully working programs should be set to public so you are able to search for it with all of the calculator's already built in functions.


4) You should have a screen that looks similar to the one below.


5) We need to delete global variables to ensure the variables we use aren't already being used. If we do not delete variables, we may run into issues later. Click your menu button then navigate to "Define variables" and select "Delete variable"





6) From here we delete six different variables. We need two instances of each variable. The first instance will be the original value provided and the second instance will be saving the mathmatical equations.


7) Now we need to actually input variables into the function. Go to menu, and navigate to "I/O," which stands for input/output, and select "Request string."


8) You will now have the type quotation marks to notify the user what you are requesting followed by a comma ',' to assign the input to a variable.


9) We will do the same for variable b1


10) We are going to define two more variable to sqaure the intial values.


11) We use := as the equals sign since that is the calculator's syntax.


12) Now that we have a and b squared; we need to add them. Their sum will equal to an instance of C therefore we are saving it to C1.


13) The final answer of C should be the radical of the sum a and b squared. We have saved the instance of this variable in c1 so we take the radical, square root, or set it to the 1/2 power since they're all the same.


14) To display the final answer, we will go to menu and to i/o again. This time we will select "Disp" which means Display. Writing out "Disp c2" will provide our answer after we run the program. Unfortunetly, I forgot to take a screenshot.


15) To run our program, we need to save it by hiting ctrl+s on our calculator. Following, we will go back to menu, navigate to Check, and select "Check Syntax & store"


16) Bummer! We hit a syntax error.


17) After some troubleshooting and further research on ti-nspire resource website, I was unable to find our erorr. In comparing the syntax they provide and ours, it looks perfect.


18) I was reluctant to give up. Another way to input variables was to simply put them when we run the program. I read this on the resource website while attempting to correct my error.


19) Doing tis means I can simply plug the variable directly into the equation using CAS (computer alegbra system) functionality. This was able to save without any errors.


20) So, how do we actually see if this works? Hit ctrl and doc to add a new page. Then select add calculator. Using Mathbootcamp's examples, we can see if our calculations align.


21) The syntax to run our function is abc(a,b) and we will replace a with the number that represents a in the equation; the same will go for b.





22) Our first test is abc(6,8) which worked to my surprise!





23) Let's test again to ensure it wasn't a fluke. This time we are using abc(3.1,2.8) which also worked!





24) That's it! You've now made your first ti-nspire cx ii cas ti-basic program! Congratulations.
